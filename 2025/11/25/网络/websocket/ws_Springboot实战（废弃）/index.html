ServerEndpointExporter 的作用是：
让 Spring Boot 自动识别并注册所有用 **@ServerEndpoint** 注解的类，让它们成为真正的 WebSocket 服务端。

WebSocket 服务核心类

两种实现方式：

  • JSR-356 是官方标准，轻量快速。
  • Spring WebSocket 是 Spring 封装版，更易扩展与集成。
对比项 JSR-356 Spring WebSocket
定义来源 Java 官方标准（Java EE 规范） Spring 自行封装的实现
包路径 javax.websocket.* org.springframework.web.socket.*
端点声明方式 使用注解 @ServerEndpoint 使用配置类注册 WebSocketHandler
生命周期注解 @OnOpen
、@OnMessage
、@OnClose
、@OnError		 方法覆写：afterConnectionEstablished()
、handleTextMessage()
等
依赖注入 不能直接注入 Spring Bean（需工具类） 可直接注入任何 Spring Bean（Service、Mapper 等）
注册机制 需在内嵌容器中配置 ServerEndpointExporter 使用 @EnableWebSocket
+ WebSocketConfigurer
扩展能力 轻量、靠自己实现鉴权/路由 可用拦截器、整合 Security、STOMP、消息代理等
适用场景 简单推送、语音转写、轻量聊天 企业级消息、聊天室、通知系统
优点 标准统一、实现简单 与 Spring 高度整合、扩展强
缺点 与 Spring 集成弱、配置麻烦 稍复杂、抽象层更多

JSR-356

常用注解

  • @ServerEndpoint(value = "/ws/chat/{userId}", decoders = {...}, encoders = {...}, configurator = ...)
  • 声明一个 WebSocket 端点；value 是 URL；可带 {} 路径参数。
  • encoders/decoders：对象与文本/二进制的转换（实现 Encoder / Decoder 接口）。
  • configurator：自定义握手、Header 读取、与 Spring 注入打通等高级定制。
  • @OnOpen
  • 连接建立时回调。常见签名：onOpen(Session s, EndpointConfig c, @PathParam String id)。
  • 典型用途：鉴权、登记在线用户。
  • @OnMessage
  • 收到消息时回调。可重载文本版 onMessage(String, Session)、二进制版 onMessage(ByteBuffer, Session)，或 onMessage(PongMessage, Session)。
  • 可加 @PathParam 取路径参数。
  • @OnClose
  • 连接关闭时回调。可拿到 CloseReason 做清理。
  • @OnError
  • 端点抛异常时回调。可记录日志/熔断统计。
  • @PathParam("xxx")
  • 绑定 @ServerEndpoint 路径里的 {xxx} 到方法参数。
