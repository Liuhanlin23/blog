一.mysql多表查询方式对比

查询类型 关键字 返回结果 典型用途 示例 SQL 注意事项
内连接（Inner Join） INNER JOIN
/ JOIN		 仅返回两表中匹配的记录（交集） 查询两个表中都有匹配关系的数据 SELECT a.name, b.course FROM student a INNER JOIN score b ON a.id = b.stu_id; 无匹配的记录不会出现在结果中
左连接（Left Join） LEFT JOIN 保留左表全部记录，右表无匹配则为 NULL 查询主表数据及其关联数据（即使没有关联） SELECT a.name, b.course FROM student a LEFT JOIN score b ON a.id = b.stu_id; 条件放在 ON
而不是 WHERE
，避免过滤掉 NULL
右连接（Right Join） RIGHT JOIN 保留右表全部记录，左表无匹配则为 NULL 与左连接相反，用于保留右表数据 SELECT a.name, b.course FROM student a RIGHT JOIN score b ON a.id = b.stu_id; 不常用，可改写为 LEFT JOIN
全连接（Full Join） FULL JOIN
（MySQL 不支持，可用 UNION
模拟）		 返回两表所有记录，匹配则合并，不匹配则补 NULL 查看两表所有数据（匹配与不匹配） (SELECT * FROM A LEFT JOIN B ON ...) UNION (SELECT * FROM A RIGHT JOIN B ON ...) MySQL 无原生支持
交叉连接（笛卡尔积） CROSS JOIN
/ 省略 ON 条件		 返回两表所有组合（A×B） 生成测试数据、枚举组合 SELECT a.name, b.course FROM student a CROSS JOIN course b; 若无 WHERE 条件，结果集极大，慎用
自连接（Self Join） 表自己连接自己 将同一张表当作两张表使用 查询层级结构、父子关系 SELECT a.name AS emp, b.name AS manager FROM employee a LEFT JOIN employee b ON a.manager_id = b.id; 必须使用表别名区分
多表连接（Multi Join） 多次使用 JOIN 同时连接三张及以上表 复杂报表查询 SELECT * FROM A JOIN B ON ... JOIN C ON ... 注意驱动表选择与索引使用
子查询（Subquery） SELECT ... WHERE id IN (SELECT ...) 嵌套查询结果再过滤 复杂条件筛选、聚合对比 SELECT * FROM user WHERE id IN (SELECT user_id FROM order); 性能较差，可改写为 JOIN
联合查询（Union / Union All） UNION
/ UNION ALL		 合并两个结果集 合并同结构查询结果 (SELECT name FROM t1) UNION (SELECT name FROM t2) UNION
去重，UNION ALL
不去重，字段数和类型需一致

笛卡尔积

笛卡尔积就是：将两个表中所有的记录逐一组合，生成所有可能的排列。

结果行数 = 表A行数 × 表B行数

内连接

内连接（INNER JOIN）本质上就是笛卡尔积，只不过通常会加上连接条件。
➤ 写了条件 ⇒ 有条件的笛卡尔积（只保留匹配行）
➤ 没写条件 ⇒ 无条件的笛卡尔积（所有组合都出现）

SQL语句

SELECT 列名列表 FROM 表1 [AS] 别名1 （INNER） JOIN 表2 [AS] 别名2 ON 连接条件;

INNER关键字可省略

左连接，右连接

  • LEFT JOIN： 保留左表全部记录，右表匹配不到的部分字段用 **NULL** 填充。
  • RIGHT JOIN： 保留右表全部记录，左表匹配不到的部分字段用 **NULL** 填充。
  • ON： 指定两表的关联条件（例如 s.class_id = c.id），用于判断哪些行是匹配的。

SQL语句

SELECT 列名列表 FROM 表1 [AS] 别名1 LEFT JOIN 表2 [AS] 别名2 ON 连接条件;

SELECT 列名列表 FROM 表1 [AS] 别名1 RIGHT JOIN 表2 [AS] 别名2 ON 连接条件;

全连接

全连接（FULL JOIN / FULL OUTER JOIN）
会返回左表和右表的所有记录，

  • 匹配的行：正常合并
  • 不匹配的行：用 NULL 填充缺失部分

换句话说：

FULL JOIN = LEFT JOIN + RIGHT JOIN 的并集

SQL语句

SELECT 列名列表 FROM 表1 [AS] 别名1 FULL JOIN 表2 [AS] 别名2 ON 连接条件;

自连接

自连接（Self Join）
是指一张表自己连接自己，用于查询同一张表中存在层级关系或关联关系的数据。

本质上就是：

把同一张表当作两张表来用，通过别名区分。

SQL语句

SELECT 列名列表 FROM 表名 AS 别名1 [INNER | LEFT | RIGHT] JOIN 表名 AS 别名2 ON 连接条件;

✅ 注意：

  • 必须给表起两个不同的别名（例如 a 和 b）；
  • 否则 MySQL 无法区分同一张表的两次引用。

联合查询

联合查询（UNION）
用于将多个 **SELECT** 查询的结果纵向合并为一个结果集。

它不是“横向连接”（那是 JOIN），
而是上下拼接，把多次查询的结果叠加在一起。

SQL语句

SELECT 列名列表 FROM 表1 … UNION [ALL]

SELECT 列名列表 FROM 表2 …;

UNION 和UNION ALL的区别

UNION 会自动去重，UNION ALL 不会去重。
