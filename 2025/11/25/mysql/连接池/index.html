  1. 在springboot+mybatis 场景中，默认连接池是HikariCP（spring boot 默认DataSource），Mybatis 只是复用它
  2. 连接池的优点
  • ** 复用连接，降低开销**：避免频繁 TCP/TLS 握手 + 认证，显著降低延迟。
  • 吞吐更稳：提前或按需维护一组可用连接，减少高峰期建连抖动。
  • 超时与容错统一：连接获取超时、校验超时、泄漏检测等集中可配，运维友好。

Mysql 的连接超时机制

  • MySQL 定义了空闲连接的最大空闲时长,由 wait_timeout 参数控制,默认八小时,如果空闲连接超过了这个时间,连接器就会自动将它断开
  • 一个处于空闲状态的连接被服务端主动断开后，这个客户端并不会马上知道，等到客户端在发起下一个请求的时候，才会收到这样的报错“ERROR 2013 (HY000): Lost connection to MySQL server during query”。

同时也要注意的是对于springboot 的默认连接池，如果在执行sql 的时候发现连接已经被sql 连接器中断，那么就会导致这个sql 执行报错不会重新连接

如何解决这个问题

  1. 可以在连接池中设置先探活,死了就丢掉重新连接
  2. 在MySql 中断连接之前,连接池主动下线旧连接

注: 关于连接池主动下线旧连接会导致连接越来越少么?

	不会 连接池内配置了一个最大 ，最小连接数量,维护了一个区间

MySQL 缓存机制

在MySQL 中 有两种缓存，一种是server 层的缓存，一种是存储引擎层的缓存Buffer Pool

  1. server 层的缓存: MySQL 收到SQL语句后会解析是不是查询,如果是查询就先去查询缓存，查询缓存是以key-value 形式保存在内存中的，key 保存的是查询语句,value 是结果集,但是这个server 层的缓存很鸡肋,因为如果只要有一个表的数据更新了,那么这个表的所有缓存都会被删掉，所以MySQL 8.0之后删除了server 层缓存,之前的版本可以通过配置关掉
  2. buffer pool 缓存： 缓存的是数据页/索引页（物理页），用来减少磁盘 I/O。
