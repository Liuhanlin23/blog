分组查询

分类于 MySQL ， SQL

MySQL 的 分组查询 主要是通过 GROUP BY 子句来实现的，它将查询结果按指定的列进行分组，通常配合聚合函数（如 COUNT()、SUM()、AVG()、MAX()、MIN() 等）来对每一组数据进行计算和统计。

基本语法

SELECT column1, column2, aggregation_function(column3)

FROM table_name

GROUP BY column1, column2;


  1. **SELECT** 用于指定查询的列，包含：
  • 普通列：如 column1 和 column2，用于分组的依据。
  • 聚合函数：如 aggregation_function(column3)，对某列（例如 column3）应用聚合操作。

2. **GROUP BY**``column1, column2：按 column1 和 column2 列分组，将数据分成多个组，每个组的值在这两列上是相同的。

3. ``aggregation_function(column3)：对列 column3 的数据执行聚合操作并返回一个结果。常见的聚合函数有：

  • COUNT(column3)：返回 column3 中非 NULL 值的行数。
  • SUM(column3)：返回 column3 所有值的和。
  • AVG(column3)：返回 column3 所有值的平均值。
  • MIN(column3)：返回 column3 中的最小值。
  • MAX(column3)：返回 column3 中的最大值。
  1. 注意

在 GROUP BY 分组查询中，聚合函数中的列不一定需要出现在 <font style="color:#DF2A3F;">GROUP BY</font> 中，但如果列没有出现在 <font style="color:#DF2A3F;">GROUP BY</font> 中且没有经过聚合函数处理，它不能直接在 <font style="color:#DF2A3F;">SELECT</font> 中查询，否则查询会报错。

例子

假设有一个 employees 表，包含 department_id（部门ID）、salary（薪资）、hire_date（入职日期）等列。

按部门分组，计算每个部门的总薪资

SELECT department_id, SUM(salary) AS total_salary

FROM employees

GROUP BY department_id;

  • 这里，department_id 是分组依据，SUM(salary) 计算每个部门的总薪资。
  • salary 并没有出现在 GROUP BY 中，但它通过 SUM() 聚合函数来处理。

按部门分组，计算每个部门的最高薪资和入职最早的员工

SELECT department_id, MAX(salary) AS highest_salary, MIN(hire_date) AS earliest_hire_date

FROM employees

GROUP BY department_id;

  • department_id 是分组依据，MAX(salary) 和 MIN(hire_date) 分别用于计算每个部门的最高薪资和最早入职的日期。
  • salary 和 hire_date 并没有出现在 GROUP BY 中，但它们通过聚合函数来处理。

错误示例（如果你没有使用聚合函数处理其他列）

SELECT department_id, hire_date

FROM employees

GROUP BY department_id;

这个查询会报错，因为 hire_date 没有使用聚合函数，且它也没有出现在 GROUP BY 中。在这种情况下，数据库不能确定如何处理 hire_date，因为每个部门可能有多个不同的入职日期。

HAVING 子句

HAVING 子句的作用

  • HAVING 用于对 分组后的数据 进行过滤。它类似于 WHERE 子句，但不同的是，WHERE 用于过滤 原始数据，而 HAVING 用于过滤 已经分组 后的结果。
  • **HAVING** 常与聚合函数一起使用，用于根据聚合结果进行条件判断。

HAVING 子句的语法

SELECT column1, column2, aggregation_function(column3)

FROM table_name

GROUP BY column1, column2

HAVING condition;


  • **HAVING** 子句在分组之后应用，用来过滤聚合结果。
  • HAVING 后面可以跟聚合函数的条件（例如 COUNT()、SUM()、AVG() 等）。
  • WHERE 不能用于过滤聚合后的结果，必须先使用 GROUP BY 进行分组，再使用 HAVING 来过滤。

HAVING 和 WHERE 的区别：

  • WHERE 用于过滤 原始数据，不能与聚合函数一起使用。
  • HAVING 用于过滤 分组后的数据，可以与聚合函数一起使用。

分组查询数据链路

假设我们有以下 employees 表：

department_id employee_id salary
1 101 5000
1 102 6000
2 103 7000
2 104 8000
3 105 9000
3 106 10000

我们要执行一个按 department_id 分组的查询，来计算每个部门的 总薪资（SUM(salary)）和 员工人数（COUNT(employee_id)），并筛选出薪资总和大于 15000 的部门。

SELECT department_id, SUM(salary) AS total_salary, COUNT(employee_id) AS employee_count

FROM employees

GROUP BY department_id

HAVING SUM(salary) > 15000;


1. 原始数据：

我们从表 employees 中根据where查询出原始数据

2. 分组操作：

根据 department_id 列将数据分成三个组。

  • 组 1: department_id = 1
    • (101, 5000), (102, 6000)
  • 组 2: department_id = 2
    • (103, 7000), (104, 8000)
  • 组 3: department_id = 3
    • (105, 9000), (106, 10000)

3. 应用聚合函数：

对每个分组应用聚合函数来计算 总薪资（SUM(salary)）和 员工数（COUNT(employee_id)）。

  • 组 1:
    • 总薪资： 5000 + 6000 = 11000
    • 员工数： COUNT(2)
  • 组 2:
    • 总薪资： 7000 + 8000 = 15000
    • 员工数： COUNT(2)
  • 组 3:
    • 总薪资： 9000 + 10000 = 19000
    • 员工数： COUNT(2)

4. 查询结果：

这时，每个分组的聚合结果已被计算出来。

department_id total_salary employee_count
1 11000 2
2 15000 2
3 19000 2

5. 应用 **HAVING** 子句：

HAVING 用于过滤分组后的结果。在这个查询中，我们只保留总薪资大于 15000 的部门。

department_id total_salary employee_count
3 19000 2

6. 最终结果：

最终，查询结果只包含总薪资大于 15000 的部门：

department_id total_salary employee_count
3 19000 2
