四大引用

分类于 Java

JAVA中有四种引用类型，分别是强引用，软应用，弱引用，虚引用

一、强引用（Strong Reference）

  • 定义：最常见的引用，使用 new 创建。
  • 特性：只要有强引用存在，对象就不会被回收；即使内存不足，也只会抛出 OutOfMemoryError。

二、软引用（Soft Reference）

  • 定义：当内存不足时，GC 会回收软引用对象。
  • 用途：常用于实现缓存。

三、弱引用（Weak Reference）

  • 定义：只要发生 GC，就会回收对象（无论内存是否足够）。
  • 用途：常用于ThreadLocal、WeakHashMap 等。

四.虚引用

引用队列（ReferenceQueue）

在理解虚引用前，先要了解 引用队列（ReferenceQueue）。
当某个引用（软、弱、虚）所指向的对象被判定可回收时，JVM 会将“引用对象本身”（而非目标对象）加入该队列。
不同的是，软引用和弱引用在对象被标记为可回收时入队，而虚引用则在对象真正被回收后才入队。

什么时虚引用

虚引用（PhantomReference） 是 Java 提供的一种最弱的引用类型。
它的最大特点是：

  • 不能通过 **get()** 访问对象（get访问返回值总为null）
  • 只用来感知对象被真正回收的时机

为什么需要虚引用

我们知道：

  • 软引用：用于缓存，内存不足时清理；
  • 弱引用：用于弱依赖对象，GC 一来就清理；
  • 但这两种引用入队太早（对象还没真正销毁）。

有些资源（比如：文件句柄、堆外内存、Socket 连接）只有在对象彻底销毁时才安全释放。
这时候就要用虚引用。
